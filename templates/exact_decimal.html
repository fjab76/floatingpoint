Exact value and d-digit decimals

Enter a number (parsed as a Python/IEEE-754 double). The server shows the exact rational decimal for that float and which d-digit decimal strings round to the same value.

Why this matters

Literals like 0.1 are not exact in binary64. The tool shows the true value and how many short decimal spellings map to it.

  • 0.1 is stored as an exact rational close to one tenth.
  • 0.1 + 0.2 evaluates to a float whose exact decimal is not exactly 0.3.
