Exact value and d-digit decimals

Enter a number (parsed as a Python/IEEE-754 double). The server shows the exact rational decimal for that float and which d-digit decimal strings round to the same value.

One float, many decimal spellings

Parsing a decimal string gives exactly one double, but the reverse is not unique: the same IEEE-754 binary64 value can be written as many different decimal literals, because conversion rounds to the nearest representable float. So the “exact” value of a float is a single rational in the machine, while humans still have more than one spelling at a given precision.

Below you choose how many significant decimal digits (d) to use; the tool then lists every d-digit decimal (in that sense) that maps to your float. That is why the count can be greater than one.

Why this matters

Literals like 0.1 are not exact in binary64. The tool shows the true value and how many short decimal spellings map to it.

  • 0.1 is stored as an exact rational close to one tenth.
  • 0.1 + 0.2 evaluates to a float whose exact decimal is not exactly 0.3.
