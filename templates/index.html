Floatingpoint

This site is a hands-on companion for understanding IEEE-754 double-precision (binary64): how values are stored, why familiar decimals like 0.1 do not have exact finite binary expansions, and how far apart neighboring floats are in each binade (exponent band).

When floating-point is appropriate: performance, hardware support, and problems where small relative error is acceptable (many simulations, graphics, signal processing).

When arbitrary precision (e.g. Decimal) matters: accounting and money, legal or compliance rounding rules, and any workflow where you must control decimal rounding or preserve exact rational quantities that binary64 cannot represent.

Explore the tools below. Each returns only the fields needed to interpret a value: representation, bits, exact rational decimal, and unbiased exponent where relevant.

  • Exact value and d-digit decimals — see the exact decimal for a float and which short decimal literals round to the same number.
  • Segment / ULP — see the exponent segment for a float and the exact spacing (ULP) between adjacent doubles in that band.
